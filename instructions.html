01

What this repo actually is

This is one n8n workflow (Video_Automation_Workflow_with_Manim.json) plus the Python engine it calls (main.py and functions_manim.py). Together, they take a narration audio file and turn it into a complete, narrated, animated video — automatically.

No stock footage. No manual keyframing. No editing timeline. The visuals are generated as code and rendered fresh for every video, which is why the channels built on this system never repeat the same clip twice.

This isn't a demo or a toy version — it's the real pipeline, with the API keys stripped out so you can drop in your own and run it yourself.

02

What is Manim?

Manim is an animation engine originally built by Grant Sanderson (the creator of 3Blue1Brown) to make math videos. Instead of dragging shapes around in a timeline editor, you write code that describes what should appear on screen and how it should move — and Manim renders that code into a video file, frame by frame.

A very simplified example of what Manim code looks like:

text = Text("34,000", font_size=140, color=GREEN)
self.play(FadeIn(text, scale=0.9))
self.wait(2)

That's it — that's an animation. This repo uses an AI model to write that code automatically based on what the narration is saying, so every chart, counter, and diagram is generated to match the script, with zero manual animation work.

03

What is n8n?

n8n (pronounced "n-eight-n") is a visual automation tool — think of it as connecting boxes on a canvas, where each box does one job (download a file, call an API, upload a video) and passes its result to the next box. It's the same category of tool as Zapier or Make, except it's free to self-host and gives you full control over the logic.

In this workflow, n8n is the conductor. It doesn't render the video itself — it triggers the Python engine (which does the actual rendering), waits for it to finish, then automatically uploads the result and posts about it.

If you've never opened n8n before: install it, open the editor, and use the Import from File option to load the .json workflow from this repo. Every box (called a "node") will appear on the canvas exactly as it was built — you're not starting from scratch.

Step by step

StageWhat happens
TranscribeThe narration audio is transcribed locally with Whisper, producing word-level timestamps.
Story BeatsGPT reads the transcript and breaks it into "beats" — small, timestamped ideas the video needs to visualize.
Manim CodeBeats are grouped into short chunks (a few seconds each), and GPT writes real Manim scene code for each chunk — a chart, a counter, a formula, whatever the beat calls for.
RenderEvery chunk is rendered independently, duration-locked to match the audio exactly, then stitched back together with ffmpeg.
Uploadn8n picks up the finished file and handles the platform upload, thumbnail, and metadata.
Why chunks instead of one long render? If a single chunk fails (bad code, a crash), only that few-second piece needs a fallback — not the entire video. That's what makes this pipeline resilient enough to run unattended.

Setup & requirements

You'll need the following installed before running anything:

  • Python 3.11+ with pip
  • n8n (self-hosted or n8n Cloud)
  • ffmpeg on your system PATH
  • Manim Community Edition (installed via requirements.txt)
  • An OpenAI API key (for the transcription-to-code steps)

Install the Python dependencies

pip install -r requirements.txt

Set up your environment file

Copy .env.example to .env and fill in your own API keys. Never commit your real .env file — it's already excluded via .gitignore.

Run the Python engine

uvicorn main:app --host 0.0.0.0 --port 8000

This starts the rendering server that n8n will call.

06

Importing & running the workflow

  1. Open your n8n editor.
  2. Click Import from File and select Video_Automation_Workflow_with_Manim.json from this repo.
  3. Every node will need its own credentials reconnected (YouTube, GitHub, image generation, etc.) — the ones in this file were stripped out on purpose before it was made public.
  4. Update any HTTP Request nodes pointing at http://localhost:8000 if your Python engine is running somewhere else.
  5. Drop your own narration audio into the Audio_Voice folder, matching the filename the workflow expects.
  6. Trigger the workflow and watch it run — transcription, beat analysis, code generation, and rendering all happen automatically.

07

Making it your own

This workflow was built for finance and math explainer content, but the underlying system isn't locked to either niche:

  • Swap the topic: the GPT prompts in main.py control what kind of visuals get generated — rewrite the system prompt for your own subject.
  • Add your own Manim building blocks: functions_manim.py is a library of reusable animation functions (counters, bar charts, comparisons, etc.) — add new ones for your content type.
  • Change the output format: resolution, frame rate, and orientation (vertical vs. landscape) are all set near the top of main.py.

08

Common issues

Whisper is slow / using CPU

If you don't have a discrete GPU, transcription runs on CPU and will be noticeably slower on longer audio files. This is expected, not a bug — a 9-10 minute narration file can take several minutes just to transcribe.

A chunk keeps failing the same way

Check the debug source saved for that chunk (written automatically on every render, pass or fail) to see the exact code GPT generated — it's usually a banned Manim class or a syntax issue that gets auto-repaired on the next attempt.

Render is taking a long time

Longer scripts mean more chunks, more GPT calls, and more individual renders. This is a genuinely CPU/API-bound pipeline — there's no shortcut around a long narration file needing more processing.
